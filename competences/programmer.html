SAÉ 1.5 - Traiter des données

Contexte

Démonstration de la capacité à récupérer des données sur un réseau, à les transformer après calculs, et à les transmettre automatiquement vers un serveur web.

Mission

Développement d'un script automatisé récupérant des données CSV sur data.gouv.fr, appliquant des traitements (moyennes, maximum, filtrage), les convertissant en JSON, et les transmettant à un site web pour affichage graphique. Gestion de projet via GitLab.

Compétences mobilisées

  • Requêtes HTTP et APIs web
  • Visualisation de données
  • Gestion de version avec GitLab

SAÉ 2.1 - Construire un réseau informatique pour une petite structure

Contexte

Dans le cadre de l'infrastructure complète d'entreprise, développement de scripts pour automatiser la gestion des services et exploitation des capacités réseau à des fins de sécurité offensive et de surveillance.

Missions
  • Scripts d'administration Docker (Bash) : développement d'un script Bash pour automatiser la gestion des conteneurs Docker (démarrage, arrêt, mise à jour des services TFTP, BDD, Web).
  • Analyse réseau offensive (Python Scapy) : script Python avec Scapy pour reconstituer une conversation VoIP interceptée sur un réseau non sécurisé. Mise en œuvre d'une attaque VLAN Hopping puis déploiement des contre-mesures.
  • Reconnaissance vidéo (Python OpenCV) : script Python utilisant OpenCV pour la reconnaissance automatique de badges avec journalisation des entrées/sorties (date, heure, nom, prénom).
Compétences mobilisées
  • Bash : automatisation et gestion de conteneurs Docker
  • Python Scapy : analyse et manipulation de trames réseau
  • Python OpenCV : reconnaissance visuelle et journalisation
  • Sécurité offensive : VLAN hopping et contre-mesures

SAÉ 2.3 - Dynamiser le portfolio avec Flask

Évolution du portfolio statique vers une application web dynamique utilisant le framework Flask, avec interaction base de données pour la gestion des compétences.

Compétences mobilisées

  • Développement web full-stack (front + back)
  • Déploiement d'application web

SAÉ 3.2 - Développer un serveur de recherche documentaire

Contexte

Développement d'un service réseau applicatif client-serveur permettant la recherche de mots dans des documents de formats variés (txt, HTML, PDF, Excel).

Mission

Développement d'un serveur Python offrant un service de recherche multi-format avec affichage du contexte (ligne trouvée, textes environnants). Respect PEP 8, découpage en fonctions simples, docstrings, constantes nommées, chemins relatifs. Projet GitLab avec README en anglais et fichier requirements.txt.

Compétences mobilisées
  • Python : architecture client-serveur, PEP 8
  • Traitement de fichiers multi-formats (txt, HTML, PDF, xlsx)
  • Documentation technique (README, docstrings)
  • Gestion de version et collaboration via GitLab

SAÉ 4.1 - Sécuriser un réseau d'entreprise inter-sites

Contexte

En quadrinôme, réalisation d'une maquette Packet Tracer d'un réseau d'entreprise sur deux sites distants avec segmentation avancée et sécurisation complète.

Mission

Conception et configuration d'une maquette avec trois réseaux LAN par site (Service / Production / Admin), tunnel IPsec GRE inter-sites, rédaction d'un Write Up complet. Spécialisation individuelle sur l'un des axes : sécurisation DNS (DNSSEC), sécurisation Web (HTTPS), tests de sécurité, ou recommandations ANSSI (checklist).

Compétences mobilisées
  • Packet Tracer : modélisation réseau avancée
  • VPN IPsec GRE inter-sites
  • DNSSEC, HTTPS, tests de sécurité ou référentiel ANSSI
  • Rédaction d'un Write Up technique détaillé
